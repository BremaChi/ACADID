# Core Platform Requests

Status: Active  
Owner: Core Platform Team  
Last updated: 2026-05-20

## Purpose

Use this file as the shared inbox for product workstreams that need Core Platform support.

This is for requests that affect:

- API contracts.
- Auth/session behavior.
- Database schema or migrations.
- Queue, worker, webhook, or background job behavior.
- Shared UI shell/navigation contracts.
- Founder Console controls.
- Security, rate limits, audit logging, or secret handling.

## How To Leave A Request

Add a new item under `Open Requests` using this format:

```md
### YYYY-MM-DD - Short Request Title

- Requesting team:
- Status: OPEN
- Priority: LOW / MEDIUM / HIGH / URGENT
- Related files:
- Related endpoint or contract:
- What is needed:
- Why it is needed:
- Blocking work:
- Notes:
```

## Rules

- Do not paste secrets, API keys, passwords, access tokens, database URLs, or private document links.
- If the request changes an API, auth, database, webhook, queue, worker, shared state, or shared UI contract, update the relevant contract document before another team depends on it.
- Prefer additive, backward-compatible changes.
- Mark a request `BLOCKED` when it needs founder/product decision.
- Mark a request `RESOLVED` only after the fix is committed and pushed.
